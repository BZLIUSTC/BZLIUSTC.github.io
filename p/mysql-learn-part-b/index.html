MySQL服务器管理规则

存储引擎

含义：mysql服务器提供表文件管理方案

查看mysql服务器存在存储引擎show engines

修改mysql服务器默认存储引擎：通过修改my.ini设置默认存储引擎 default-storage-engine=myisam

查看表文件依赖存储引擎show create table 表文件

修改表文件依赖存储引擎alter table表文件 engine=新存储引擎名

INNODB与MYISAM 区别

  1. myisam:

    • 采用三个文件存储信息 【xx.frm —- 字段信息、xx.myd — 数据行、xx.myi —- 索引】
    • 增加表文件操作速度,不支持事务管理
    • 如果表文件操作以查询为主,建议采用myisam

  2. innodb:

    • 采用frm文件存储信息（字段,数据行,索引）
    • 增加表文件数据安全性。在九种存储引擎中唯一支持事务管理

约束管理方案

确保字段内容符合实际情况。

MySQL有五种约束：

非空约束

不允许字段值为NULL。格式create table tb(字段名 字段类型 not null)

唯一性约束

字段值不能重复，但是可有多个NULL。格式create table tb(字段名 字段类型 unique)

主键约束

用于管理主键字段，不能重复也不能为NULL。格式create table tb(字段名 字段类型 primary key)

外键约束

用于管理外键字段，要求字段值必须来自于一方表主键字段已经存在的值，同时允许NULL。 格式alter table 多方表 add constraint 约束对象名 foreign key(多方表外键字段名) references 一方表(一方表主键字段名)

例如，为emp添加了约束：alter table emp add constraint extkey_deptno foreign key(deptno) references dept(deptno)， 那么就无法执行如下添加：insert into emp(empno,ename,deptno) values(555,'Nagisa',50)

视图（背 @@@）

视图是MySQL提供的内置管理对象。 视图对象内部采用 键值对 的形式存储，key 存储查询语句，value 存储查询语句对应字段管理权.

这是由于，大项目中一个数据库可服务于多个业务，要对不同业务访问数据库做权限管理。

例子：视图一对象

视图的创建：create view 视图对象名 as 查询语句，例如create view view_one as select empno,ename,job,sal from emp

视图的作用：

  1. 提高查询语句的复用率。举例：可以直接使用select * from view_one，直接得到select ename,job,sal from emp的结果。 这有利于复用复杂的查询语句
  2. 分发开发者对表的使用权力。举例：
    • 插insert into view_one(empno,ename,job) values(7766,'MIO', 'Manager')注意1
    • 删delete from view_one where ename='Nagisa'，
    • 改update view_one set ename='BoBB' where ename='AliceWang' 避免开发者直接操作整个emp表的所有字段

索引 index – SQL优化（必考 @@@）

什么是索引

  1. 索引存在硬盘中一个数组
  2. 数组有序存放表文件中某个字段下所有数据以及数据所在行数
  3. 如果表文件采用INNODB引擎管理,索引存在frm文件；如果表文件采用myisam引擎管理,索引存在myi文件
  4. 索引作用: 减少遍历临时表中数据行次数,增加查询速度【但是增删查会影响速度，因为要更新索引】

例如参考脚本中，为年龄字段构建索引。 一般来说，where age=20定位条件要遍历整张表，若有索引则会使用Btree算法

索引的操作：

  • 查看已经存在的索引：show index from 表名；MySQL自动为主键、外键约束设置索引
  • 创建索引：create index 索引名 on 表名(字段名)
  • 删除索引：drop index 索引名 on 表名

执行计划

作用：查看一条查询语句是通过 索引还是遍历 来获取数据行
格式：explain 查询语句，查询结果中的 type 代表运行速度（all 最慢），rows 为实际遍历行数

type 有几种取值：

  • all：最烂，没有通过索引定位数据行而是全表扫描。SQL优化要避免存在 all 级别的语句；
  • index：比较烂，采用全表扫描定位数据行，但select数据抓取是从索引中得到的（几乎没优化）。SQL优化要避免存在 all 级别的语句；
  • range：还行，采用索引提供数据行位置，同时一次返回多个数据行。range级别并不稳定，如果通过索引返回的行数 > 总行数三分之一，则MySQL（处于性能考虑）会放弃使用索引。SQL优化的最低标准；
  • ref：理想，采用索引提供数据行位置，每次返回一行数据。SQL优化的最高标准；
  • const：最快，采用聚簇索引提供数据行位置

聚簇与非聚簇索引
非主键字段的索引是非聚簇索引；主键字段的索引是聚簇索引。
聚簇索引的 value 不再是行号，而是完整的一行数据，因此节省了通过行号读内存数据的时间（读索引这一步就完成了）

索引失效

  1. 如果通过索引定位行数达到了总行数1/3时,必然导致索引失效
  2. 如果在索引字段进行数学运算,必然导致索引失效
  3. 如果在索引字段进行函数处理,必然导致索引失效
  4. 前置模糊查询可以达到range；后置模糊查询和包含模糊查询一定会导致索引失效
  5. 使用MySQL自动类型转化，必然导致索引失效

事务管理 Transaction

什么是事务
对于采用innodb管理的表文件,在进行数据修改时(insert,update,delete) mysql服务器自动对当前表文件中数据进行一次备份然后再进行操作。这个备份就是事务

事务作用
在操作之后,如果需要反悔。可以通过事务覆盖表文件实现操作撤销

事务管理方式

  1. mysql服务器自动管理事务，如果当前DML命令无法正常执行。此时mysql服务器使用事务撤销本次操作。如果当前DML命令正常执行,mysql服务器销毁掉事务
  2. 手动事务管理方式,要求mysql服务器将多个SQL命令产生备份，交给同一个事务对象管理，操作完毕后由开发人员根据实际情况决定是否测校

手动事务管理格式：

事务的使用原则 ACID

Atom 原子性：只有来自于同一个业务中的SQL命令才应该交给同一个事务管理对象进行管理。

例如，撤销部门20并删除其下员工信息，则可归为同一事务； 删除一员工信息，同时新增一部门，这不是一件事情，不应该用同一事务管理。

1
2
3
4

start transaction;
delete from dept where deptno=20;
delete from emp where deptno=20;
commit; /* rollback; */

Consistency 一致性：一个多分支业务只要有一个分支执行失败或反悔（rollback），则就应该将事务中所有sql操作视为无效。

Isolation 隔离性：两个事务对同一表的操作应当彼此互相影响（与多线程有关）

Durability 持久性：commit 后无法恢复

字段值的数据类型

整型

用 int 即可

小数

  • float, 7位有效数字
  • double, 15位有效数字
  • decimal(M, D)，定点小数，以字符串存储，主要用于货币计算

字符串（常考 @@@）

char(m)：固定不可变字符串。意思是可以接受的字符个数固定，赋值时不能超过这个个数。磁盘上存贮值的空间是固定的。

例如，字段类型 char(5)，则不论存什么内容，都占那么多空间。 若存入'abc'，则内存中是'abc '；但是，SQL读取char类型数据时会自动把空格抹掉

varchar(m)：固定不可变字符串。意思是可以接受的字符个数固定，赋值时不能超过这个个数。磁盘上存贮值的空间是可变的。 存入合法的不同长度的值，占用存储大小不一样。

JDBC 通信规范

jar包基础知识

  1. 以.jar结尾文件。类似于压缩文件
  2. jar包存放class文件,用于工程与工程之间类文件复用

jar包的分类

  1. 第一方jar包:SUN公司提供JDK.jar
  2. 第二方jar包:开发人员自行生成jar
  3. 第三方jar包:其他人或则组织提供jar

接口与jar包开发形式

  1. 开发人员不需要关心工具类实现细节
  2. 开发人员只需要根据固有步骤组装工具类即可完成开发任务

JDBC通信规范介绍

  1. JDBC通信规范是SUN公司提供通信规范,用于让Java应用程序与关系型数据库服务器进行通信标准
  2. SUN公司为JDBC通信规范提供一套接口,存在JDK.jar中java.sql包
  3. JDBC通信规范涉及的接口实现类由关系型数据库服务器厂商提供以jar包形式对外输出。
  4. 普通的Java应用程序工程师只需要将指定jar包引入到工程,并按照JDBC规范指定开发流程进行命令组装即可实现与数据库服务器通信

JDBC规范中接口介绍

  1. java.sql.Connection：指定连接通道管理规则
  2. java.sql.PrepareStatement：指定数据通道“交通工具”管理规则
  3. java.sql.ResultSet：指定临时表操作规则
  4. java.sql.DriverManager工具类：负责在java类与数据库服务器之间建立连接通道

JDBC开发步骤

  1. 建立连接通道 – DriverManager

  2. 建立交通工具 – PreparedStatement

  3. 由交通工具负责将SQL命令推送到数据库服务器，服务器返回结果由交通工具带回发送请求的java类中

  4. 销毁通信涉及所有资源（交通工具、连接通道），以节省资源

使用JDBC协议连接到MySQL需要用到MySQL提供的jar包： 下载链接，注意主版本号要与MySQL主版本号对应。 直接下载。 将jar包复制到Module目录下，右键 Add as Library 即可添加。

使用JDBC访问数据库

DML 三条命令

模板如下，遵循：

  • DriverManager建立通道对象A（需要参数：url，用户名和密码）
  • 对象A建立PreparedStatement交通工具对象B（需要参数：SQL命令）
  • 对象B执行方法，从而向数据库服务器发送请求并接受返回内容（调用恰当的方法，DML要用executeUpdate()，DQL要用executeQuery()）
  • executeUpdate()返回受影响的行数，无异常则执行成功
  • 想知道对象方法返回类型，就用ctrl查看接口的定义即可

package com.foo.test;
import java.sql.Connection;
import java.sql.DriverManager;
import java.sql.PreparedStatement;

// DML命令，insert, delete, update 命令的模板即如下,
// 只需要改sqlCommand 即可
public class CmdInsertProc {
    // throw Exception 必须加, 否则getConnection报错无法处理
    public static void main(String[] args) throws Exception {
        // 命令
        String sqlCommand = "insert into dept values(60, 'ASSIST', 'SHANGHAI')";
        // 基本信息
        String userName = "root";
        String passwd = "pb23000061";
        String url = "jdbc:mysql://localhost:3306/sandbox";
        // 1. 建立通道
        Connection con = DriverManager.getConnection(url, userName, passwd);
        // 2. 建立交通工具
        PreparedStatement prep = con.prepareStatement(sqlCommand);
        // 3. 发送和收回
        int rowsAffected = prep.executeUpdate();
        System.out.println(rowsAffected + "行受影响");
        // 4. 销毁资源, 不要搞错顺序
        if(prep != null) {prep.close();}
        if(con != null) {con.close();}
    }
}
